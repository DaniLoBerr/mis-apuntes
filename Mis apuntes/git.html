¿Qué es GIT?

GIT es un sistema distribuido de control de versiones, es decir, un sistema que nos permite tener un historial completo de todo el código desarrollado sobre una aplicación. Este sistema nos permite trabajar el código de manera descentralizada, lo que quiere decir que cuando varios desarrolladores están trabajando sobre una misma aplicación, no dependen de un servidor central para acceder a los otros cambios que han ido realizando los demás, para actualizar sus propios cambios ni tan siquiera para acceder al mismo proyecto. Con GIT, cada desarrollador tiene una copia completa de todas las versiones en su máquina y tiene la opción de subir los cambios que han realizado a un servidor central (para sincronizar proyectos) el cual no es tanto problema si falla ya que cada desarrollador tiene una copia del proyecto completo en su máquina.

Usos de GIT

  • Historial de nuestro código y el de los demás.
  • Almacenamiento del código.
  • Facilidades para el trabajo en equipo.
  • Fácil acceso a un posible error en una determinada versión del código o fase del proyecto.

Flujo de trabajo de GIT

Computador -> Stage -> Commit - > Server

Computador
Proyecto con sus versiones almacenador en nuestro ordenador.
Stage
Etapa intermedia en la que seleccionamos qué archivos o qué cambios queremos añadir al repositorio.
Commit
Etapa de confirmación de los archivos o cambios del proyecto que queremos añadir al repositorio.
Server
Etapa opcional en la que pasamos lor archivos o cambios elegidos a un servidor eterno/nube/aplicación tipo Github.

GIT en terminal

Comandos de configuración

git --version
Nos indica la versión de GIT que tenemos instalada en la máquina.
git config --global
Sirve para que los cambios en la configuración que vayamos a realizar se ejecuten de manera global y no en un solo proyecto en particular.
git config --global user.name
Sirve para elegir nuestro nombre de usuario en GIT. Si el nombre del usuario lleva algún espacio entre medias, se debe escribir el nombre completo entre comillas dobles.
git config --global user.email
Para configurar nuestro email en GIT.
git config --global core.editor
Sirve para configurar nuestro editor de texto por defecto para GIT. Para indicar a git que, por ejemplo, nuestro editor de texto por defecto es VS Code, se debe escribir este comando seguido de code --wait. Con el --wait, le indicamos a la terminal que se quede esperando a que cerremos nuestro editor de texto elegido para efectuar los cambios en la configuración. Una vez hecho esto, si escribimos el comando git config --global -e, podremos comprobar nuestro archivo de la configuración global de nuestro editor. Si todo salió correcto, este archivo se nos abrirá en VS Code y veremos los datos que acabamos de actualizar.
git config --global core.autocrlf true/false
Los sitemas de Windows y Unix agregan diferentes caracteres especiales cada vez que se agrega un salto de línea en el código para marcar dicho salto de línea.
  • Windows utiliza CR (Carriage Return / Retorno de carro) y LF (Line Feed / Salto de línea).
  • Linux/Mac utiliza solo LF.
Por lo tanto, cuando dos personas se encuentran trabajando sobre un mismo repositorio pero un sus PCs tienen diferentes SOs, se necesita configurar GIT con este comando.
  • El usuario que utilice Windows, necesitará eliminar el caracter CR de su código cada vez que desee subir su código al repositorio y añadirlo cuando desee descargárselo. En este caso, el valor que hay que utilizar para este comando es True.
  • Por el contrario, un usuario que utilice Linux o Mac no agrega este caracter CR, por lo que GIT en este caso no debería realizar ninguna acción sobre el código. Pero si por alguna razón nuestro editor de texto utiliza este tipo de caracter o si el usuario lo agregó manualmente por el motivo que sea, GIT debe eliminarlo, por lo que utilizaríamos el valor Input.
git config -h
Nos muestra un listado con todos los comandos necesarios para realizar diferentes tipos de configuración en GIT.

Comandos básicos de BASH

ls
Sirve para mostrar una lista de todos los directorios que se encuentran en el directorio principal de la máquina (carpeta de usuario).
pwd
Muestra la ruta del directorio en el que nos encontramos.
cd + nombre de directorio
Sirve para entrar en uno de los directorios contenidos en el directorio en el que nos encontramos. Si el nombre del directorio consta de dos o más palabras separadas, debe escribirse entre comillas.
cd + ..
Sirve para movernos a una carpeta anterior de la ruta en la que nos encontramos. Con ../.. iríamos dos carpetas hacia atrás y así sucesivamente.
mkdir + nombre
Sirve para crear un directorio. El nombre del directorio nuevo debe escribirse entre comillas si consta de dos o más palabras separadas.
git init
Sirve para iniciar un proyecto nuevo de GIT. Se nos crea un directorio oculta llamada .git en la que nos aparecen una serie de archivos y directorios utilizados en git para poder gestionar los proyectos (detalle de implementación). Aquí es donde se almacenarán las distintas versiones del código del proyecto, las distintas ramas, los commit, etc. Esta carpeta .git suele ignorarse siempre en todos los repositorios que se vayan a crear, no se pasa al servidor central ni se comparte entre desarrolladores.
ls -a
Muestra las carpetas y archivos ocultos.

Comandos de trabajo GIT

code .
Se utiliza para abrir nuestro editor de texto VSCode con un proyecto abierto en la carpeta en la que nos encontremos en la terminal. Code indica que queremos que se abra el editor y el punto indica que queremos que se abra en la carpeta en la que nos encontramos.
git status
Nos indica el estado actual del repositorio en que nos encontramos: La rama en la que nos encontramos, lo commits que tengamos pendientes de confirmar, los cambios que tengamos pendientes de confirmar de los archivos que tengamos en seguimiento y los archivos que tengamos en el directorio del repositorio sin seguimiento.
git add
Comando utilizado para seleccionar los archivos del directorio del proyecto que queremos seguir con git. Se puede relacionar de muchas maneras, por ejemplo:
git add (nombre de un archivo.extensión)
Se selecciona un archivo en concreto
git add .(extensión)
Seleccionaríamos todos los archivos que tuvieran una extensión en concreto.
git add .
Se seleccionan todos los archivos que se encuentren en el directorio. No es recomendable utilizarlo si no se está seguro del todo ya que podríamos no darnos cuenta de haber realizado cambios en archivos que no queremos añadir o agregar cualquier tipo de archivo archivo que nos ocuparía espacio en el repositorio innecesariamente o que directamente no nos damos cuenta de que lo tenemos ahí guardado y no lo queremos añadir.
Este comando también se utiliza cuando queremos eliminar un archivo (no tengo muy claro como).
git commit -m "comentario sobre lo que se está comiteando"
Con este comando, se comprometen los cambios/archivos previamente seleccionados y, además, se agrega un comentario sobre los mismos. Se recomienda hacerlo de esta manera, aunque también se puede escribir "git commit" a secas. De esta manera, se nos abriría una ventana con el editor de cógigo predeterminado que tenemos configurado con un texto que nos dice que escribamos el mensaje sobre los cambios que estamos comprometiendo, el archivo, la rama en la que estamos,etc.
rm (nombre de archivo.extension)
Este comando se utiliza para elminar un archivo de nuestro repositorio de git. Al igual que para añadirlo, el proceso de eliminación debe pasar por las mismas fases. Una vez utilizamos este comando rm y miramos el status, vemos que el archivo se encuentra seleccionado. Y seguimos el mismo proceso que al añadirlo: git add para pasarlo a la fase de stage, git commit -m "" para comitearlo y escribir un mensaje.
git rm (nombre de archivo.extension)
Este comando nos permite ahorrarnos el paso intermedio del anterior a la hora de eliminar un archivo del repositorio. Se selecciona el archivo que queremos eliminar del repositorio y se elimina directamente.
git restore --stage (archivo.extensión)
Este comando se utiliza para sacar un cambio/archivo que previamente hemos pasado a la etapa de stage.
git restore (archivo.extension)
