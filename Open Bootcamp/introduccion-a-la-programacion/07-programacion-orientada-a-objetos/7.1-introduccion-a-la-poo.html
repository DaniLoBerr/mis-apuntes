Introducción a la programación orientada a objetos

¿Qué son los objetos?

Son entidades que representan objetos del mundo real en nuestro programa informático. Tienen una serie de propiedaddes y son capaces de realizar una serie de acciones denominadas métodos, que al fin y al cabo son las funciones que se encuentran dentro de una clase.

Los objetos se declaran de formas diferentes según el lenguaje de programación: mediante funciones dentro de estructuras en Go por ejemplo o mediante clases en Java, Python, PHP, C++, etc.

Las clases están formadas por propiedades (que son variables) y métodos (que son funciones). En Java, los objetos se crean instanciándolos a partir de las clases. Instanciarlo significa que nosotros creamos una zona de la memoria a través de una variable que tendrá las mismas propiedades y métodos que la clase de la que proviene. Para crear una instancia, primero se escribe el nombre de la clase y luego del nombre de la variable que queramos que tenga ese objeto. Escribir el nombre de la clase funciona como un tipo de dato, es decir, cuando declaramos una nueva variable, tenemos que indicar de que tipo de dato será, y este caso hacemos lo mismo con la clase. Depués se escribe la palabra "new" para indicar que queremos crear un nuevo objeto de la clase x.

Se pueden crear varias variables de una misma clase y, aunque tengan las mismas propiedades y mismos métodos, son variables independientes las unas de las otras (se guardan en zonas diferentes de la memoria).

Constructor de clases

Un constructor es una función que se ejecuta cada vez que se instancia un objeto de una clase. Los constructores de una clase son métodos cuyo nombre coincide con el nombre de la clase, y se caracterizan por no tener tipo de retorno, ni siquiera void.

Cuando no se crea un constructor, Java lo crea automáticamente, pero en el momento en que construimos un constructor, Java ya no creará constructores, sería necesario construir el resto cada vez que queramos crear una clase.

Es lo primero que se ejecuta cuando se ejecuta una instancia de una clase.

Un constructor es una forma de inicializar las propiedades de una clase cuando la instanciamos. No devuelven ningún tipo de dato (por lo que no hay que escribir int o void, etc). Se escriben dentro de una clase. Su nombre debe ser idético al nombre de la clase. Puede tener parámetros. Tiene un cuerpo donde incluir funciones.

El constructor se ejecuta cada vez que creamos un nuevo objeto. La clase se instancia, el objeto se crea instanciando su clase. El constructor se ejecuta cada vez que se ejecuta la línea instanciada.

Como decíamos, al constructor se le pueden dar unos parámetros, que serán unas variables propias de ese constructor que solo existirán para los objetos que se creen a partir de él. Los valores de esos parámetros se indican cuando se instancia la clase para ese objeto.

Sobrecarga de constructores (constructor overloading)

Consiste en que yo puedo tener dos funciones con el mismo prototipo, el mismo tipado, pero con parámetros diferentes. Puedo crear dos constructores en una misma clase en los que uno acepte parámetros y el otro no. Los objetos que se instancien a partir de ellos tendrán la misma sintaxis, pero a la hora de hacerlo, si yo a la función le doy unos parámetros estaré llamando al constructor de los parámetros y si no se los doy, estaré llamando al constructor sin los parámetros.

El constructor es lo primero que se ejecuta cuando se instancia una clase y es el lugar más propicio para inicializar las propiedades de una clase.

También hay funciones que se pueden sobrecargar en el tipo de dato, es decir, que se pueden llamar a dos funciones con los mismos parámetros y que realicen la misma acción pero que acepten y/o devuelvan tipos de datos diferentes.

Cuando tenemos un constructor con parámetros y queremos inicializar variables internas con las mismas propiedades de la clase, debemos darle el mismo nombre de las propiedades a los parámetros. Pero en este caso nos quedaría que las variables de las propiedades de la clase y las variables internas del constructor se llamarían igual, por lo que para diferenciarlas tendríamos que ponerle "this." delante de la variable que hace referencia a las propiedades de la clase (ejemplo en intellij). Con ello le decimos al programa que queremos que las propiedades de la clase, dentro de esta función, tengan el mismo valor que el que le estamos indicando en los parámetros de la función.
