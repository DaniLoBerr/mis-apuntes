Estructuras/Sentencias de control

Serie de órdenes/instrucciones que permiten modificar el flujo de ejecución de un programa. Pueden ser secuenciales, iterativas o de control avanzadas. Se basan en la utilización de condiciones para comparar situaciones y que el programa ejecute una u otra opción según la situación. Cuando se evalua una condición, solo se pueden dar dos resultados (lógica booleana). Con las estructuras de control se puede: - De acuerdo con una condición, ejecutar un grupo u otro de sentencias (If-Then-Else) - De acuerdo con el valor de una variable, ejecutar un grupo u otro de sentencias (Switch-Case) - Ejecutar un grupo de sentencias solo cuando se cumpla una condición (Do-While) - Ejecutar un grupo de sentencias hasta que se cumpla una condición (Do-Until) - Ejecutar un grupo de sentencias un número determinado de veces (For-Next)

Tipos de condiciones

  • Condiciones lógicas
    Y
    Comparación en la que se deben cumplir dos o más situaciones para que las ordenes se ejecuten. Habitualmente, en informática, se representa como &&.
    O
    Comparación en la que se debe cumplir o una u otra o ninguna de las situaciones para que las ordenes se ejecuten. Habitualmente, en informática, se representa como ||.
  • Condiciones comparativas
    • Mayor que (>)
    • Menor que (<)
    • Mayor o igual que (>=)
    • Menor o igual que (<=)
    • Igual a (==)
    • Distinto de (!=)

40 mayor que 30 => cierto 40 menor que 30 => falso (40 mayor que 30) y (30 menor que 50) => cierto y cierto => cierto (40 mayor que 30) o (30 mayor que 50) => cierto o falso => cierto

Estructura de control de selección

Condicional simple: if / else if / else

Si se cumple la condición o las condiciones (if), se ejecutan todas las acciones indicadas en el cuerpo de la sentencia. En caso contrario, si se cumple otra condición (else if), se ejecutan las acciones contenidas en esa sentencia. En cualquier otro caso (else), si no se ha cumplido ninguna de las condiciones anteriores, se ejecutarán las de esta última sentencia, denominada fallback. El fallback es la acción última por defecto, la acción que se tiene que hacer porque, por circunstancias del código, se debe realizar alguna acción.

No es necesario escribir toda la sintaxis completa, se puede utilizar solo un if por ejemplo. Lo importante es saber que, a nivel lógico, se puede tener una condición, y esa condición se puede evaluar de forma consecutiva: si pasa esto, haz esto; si pasa esto otro, haz esto otro; y, en cualquier otro caso, haz esto último.

Estructuras de control iterativas o bucles

  • while
  • do while
  • for
  • switch

Un bucle en programación es un fragmento de código que se va a ejecutar repetidamente mientras se cumpla cierta condición. Una interación es una de las repeticiones del bucle.

Bucle while

Este es un bucle tipo "mientras", mientras se cumpla cierta condición, se ejecutarán las acciones contenidas en el bucle. Constantemente se está verificando si se cunple la condición para ejecutarse. Ejemplo de psudocódigo: Var contador = 20 Mientas (contador mayor que 0), entonces: Resta 5 al contador

Curiosidad: El lenguaje "GO" no tiene esta sentencia.

Bucle do while

Este bucle es parecido al while pero con la diferencia de que en el while no siempre se cumplen las acciones de la sentencia, se realizan mientras se cumpla la condición; pero, en los bucles do while, primero se ejecuta la acción y depués se analiza la condición para ver si se ha de realizar más veces. La acción contenida en la sentencia se ejecuta, como poco, una vez. Var contador = 10 Haz Resta uno al contador Mientras (contador sea mayor que 10)

Pocos lenguajes tienen esta estructura porque realmente se podría hacer lo mismo con un while pero colocando el cuerpo de la sentencia fuera, es decir: Var contador = 20 Resta 5 al contador mientras (contador mayor que 0)

Bucle for

Es una especie de mezcla de todos los anteriores. Consta de 3 partes: inicialización, comparación y acción. Las sentencias de este tipo funcionan de la siguiente manera: Mientras se cumpla la parte de la inicialización, si se produce la parte de la comparación, realiza las acciones que se encuentran contenidas dentro de la sentencia y después ejecuta la parte de la acción. En algunos lenguajes las tres son obligatorias de indicar, en algunos ninguna. para (var contador = 10 ; contador mayor que 0 ; resta uno al contador ) imprime el valor de la variable contador Lo que hace esta sentencia del ejemplo es que "mientras que la variable contador sea igual a 10, si la variable contador es mayor que 0, imprime el valor de la variable contador y resta uno a la variable contador". Es importante saber eso, que la parte de la acción se ejecuta después de ejecutarse el código contenido en el cuerpo de la sentencia.

Normalmente, los for se utilizan cuando se quieren recorrer arrays o tipos de datos compuestos, mientras que los while se suelen utilizar para comparaciones cierto/falso. Digamos que el while es nada más que la parte de la comparación del for. Ejemplos en pseudocódigo de esto: de un while y de un for: var temperatura = 14; mientras (temperatura menor que 15) poner_abrigo var valores = |10|20|30|40|50| para (posicion_en_array=0; posicion_e_array menor que longitud_del_array; posicion igual a posicion mas uno) imprime el valor de la posicion actual en el array

En los bucles, es habitual encontrarse la variable "i", que es la que se suele utilizar para indicar la posición actual en la que nos encontramos dentro del bucle. Es una convención. (i viene de index, índice). (El valores.length nos indica cual es la longitud del array (propio de Java)).

Bucle swicht

Interruptor. Parecido al if. Bucle tipo "discierne" que se aplica sobre una condición. La diferencia entre usar un if/else if/else o un swicht depende un poco del gusto o de la guía de estilo. El profesor en este caso utiliza utiliza un switch cuando necesita utilizar más de dos else if, aunque hay que tener en cuenta que a veces no se pueden utilizar muchas comparaciones en un bucle switch. Es un bucle especialito.

Si utilizamos un bucle switch y se cumple la primera condición del mismo, pasaría una cosa en la mayoría de los lenguajes de programación que tienen la funcionalidad de switch, y es que se ejecutarían todas las acciones de todos los casos hasta el final del bucle. Para controlar esto, es necesario utilizar la palabra "break" (en Java, creo), que es una pabalabra reservada, después de cada caso, para romper la ejecución. Después del por defecto no haría falta. Con este bucle, no se realizaría ninguna acción de ningún caso hasta que se cumpliera la condición de alguno; y, a partir de él, se realizarían todas las acciones hasta el final del bucle si no se rompe el flujo.
